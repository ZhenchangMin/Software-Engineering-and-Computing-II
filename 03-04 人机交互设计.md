# 03-04 人机交互设计

> 软件工程与计算 · 人机交互模块 · 南京大学 · 刘钦
> **核心三问**：① 什么让界面变得可用（Usable）？ ② 如何从人的因素推导出设计决策？ ③ 如何选择合适的交互风格？

## PART 1：什么是 HCI 设计

> 用户依据**界面**而非功能判断系统好坏；设计不当的界面可能引发灾难性错误甚至软件被弃用。
> Joel Spolsky："当程序的行为完全符合用户的预期时，用户界面就是好的设计。" 好的界面应该是**透明的**——用户忘记自己在使用计算机。

### 可用性（Usability）的五个维度

| 属性 | 英文 | 含义 |
|------|------|------|
| 易学性 | Learnability | 短时间内掌握操作 |
| 效率 | Efficiency | 熟练用户高效完成任务 |
| 易记性 | Memorability | 中断后无需从头学习 |
| 容错性 | Errors | 错误少，能快速恢复 |
| 满意度 | Satisfaction | 使用时感到愉悦 |

> **学习曲线权衡**：易学性（新手）与效率（专家）存在冲突，不可兼得，需平衡。

## PART 2：Human（人）

### 记忆与人的特性

> 人**识别**事物的能力远强于**回忆**——这是 GUI 胜过命令行、用图标而非名称的根本原因。人对图像的记忆优于文字。

人的特性：短时记忆有限（7±2，系统应替用户记住信息）、人会犯错（设计应帮助避免错误、提供恢复机制）、人各不同（交互偏好不同）。

### Fitts' Law（菲茨定律，1954）

> 预测指向性运动（点击目标）所需时间的经验公式：**目标越大、越近 → 点击越快**（T 与 D/W 相关）。

| GUI 设计启示 | 说明 |
|------|------|
| 重要按钮要大 | 增大 W，降低误操作 |
| 菜单栏/Dock 放屏幕边缘 | 边缘等效无限宽（D/W→0） |
| 危险操作按钮小且远 | "删除"远离"保存" |
| 上下文菜单在光标处 | D=0，点击最快 |
| 触屏最小点击区 | ≥44pt(iOS)/48dp(Android) |

### 三类用户与 UX 工具

| 用户类型 | 设计侧重 |
|------|------|
| 新手用户 | 易学性、图形界面、避免出错 |
| 熟练用户 | 易学性与效率折中 |
| 专家用户 | 效率、命令行、快捷键、热键 |

- **Persona（用户画像）**：基于研究数据构建的虚构典型用户，将抽象"目标用户"具象化（设计时问"张明会怎么做？"）。
- **User Journey Map（用户旅程图）**：以时间轴展示用户体验路径（发现→首次使用→深度使用→遇到问题→推荐），记录行为/情绪曲线/痛点/机会点。

> **精神模型（Mental Model）**：发现用户对任务的精神模型，关注内在隐喻（Metaphors），找出用户真正的目标（Goal）。映射越直观，用户越不易出错。

**人因学（Ergonomics）**：物理人因学（人体测量、姿势）、认知人因学（认知负荷、决策）、组织人因学（团队协作）。三要素：使用者、设备、使用环境。

## PART 3：Computer（计算机）

输入设备（键盘、鼠标、手写笔、语音、隔空手势）/ 输出设备（显示屏、喇叭、触觉反馈）。

**移动端 HCI 特殊考量**：
- **拇指区域（Thumb Zone）**：舒适区（屏幕中下部）放高频操作 → 重要操作下移（底部导航栏的设计依据）。
- 触屏约束：最小点击 ≥44pt/48dp、手势冲突、单手优先、上下文噪声（强光需高对比度）、网络不稳定需离线反馈。
> 移动端占全球 Web 流量 60%+，**移动优先（Mobile First）**已成默认策略。

**五种界面类型**：批处理（无交互）、命令行（一维交互）、全屏（表单/菜单）、图形化（窗口/菜单/鼠标，适合不熟练用户）、多维交互（声音/视频）。

> 可视化设计要点：按任务模型设计界面隐喻，**不要把软件内部构造机制暴露给用户**。

## PART 4：Interaction（交互）

> 交互是**双向**的：用户→系统（请求/输入），系统→用户（响应/主动告知）。

### 五种交互风格

| 风格 | 典型控件 |
|------|------|
| 直接操纵 Direct manipulation | 窗口、图标 |
| 菜单选择 Menu selection | 下拉/弹出菜单 |
| 表单填写 Form fill-in | 输入框、复选框 |
| 命令语言 Command language | 终端、Shell |
| 自然语言 Natural language | 语音助手、聊天机器人 |

> **直接操纵**优点：用户感到掌控、学习时间短、操作即时反馈；问题：信息空间大时导航难、界面复杂。
> **自然语言界面**：LLM 的出现使其重新成为热点，语音助手已成主流。

**交互事件归一**：多种输入方式（触屏单击、鼠标双击、键盘 Enter）下应保持一致的交互语义，降低学习成本。

### 导航与反馈

> 好的导航像好的餐厅菜单——帮用户快速找到任务入口，且入口符合精神模型。分**全局结构**（按任务模型组织功能）和**局部结构**（界面布局视觉线索）。

**反馈时间准则（Shneiderman 2003）**：打字/光标 50~150ms；简单任务 1s；普通任务 2~4s；复杂任务 8~12s。响应时间 >15s 具有破坏性。

### 协作式设计

> 人的因素相对固定，需让计算机更多适应人——这是**以用户为中心**的根本原因。三方向：
- **低出错率**：灰色屏蔽不适当选项、错误时给建设性指导。出错信息四原则（Shneiderman）：清晰语言、精炼准确、建设性帮助、友好态度。
- **易记性**：减少短期记忆负担、逐层递进展示、直观快捷方式、有意义的缺省值。
- **视觉风格**：色彩、字体、图标、插画、布局（对比度经人因实验验证）。

## PART 5：HCI 设计过程

> 人机交互设计**贯穿需求开发与软件设计全过程**，而非最后一步。
> **对话结构图（Dialogue Structure）**：描述界面状态（节点）与用户操作（边）之间的关系。
> **原型化**：线框图 → 可交互原型，逐步验证、与用户共同评估修正。

## PART 6：黄金原则

**Pressman 三条黄金原则**：① 让用户掌控系统 ② 减少用户记忆负担 ③ 保持一致性。

**Nielsen 10 条可用性启发原则（1994）**（启发式评估核心检查单）：①系统状态可见 ②与现实匹配 ③用户可控(撤销/重做) ④一致性与标准 ⑤防止错误 ⑥识别优于回忆 ⑦灵活高效(快捷方式) ⑧简约设计 ⑨帮助恢复错误 ⑩帮助与文档。

## PART 7：GUI 设计实践

> **GUI 是首先要考虑的事情——而不是最后！** 迭代流程：线框图 → 低保真原型 → 用户测试 → 高保真原型 → 用户评估 → 最终实现。

常见控件分类：导航类（顶部导航≤7项、侧边栏、底部 Tab≤5项、面包屑）、操作类（主/次按钮、FAB、对话框）、反馈类（Toast、Progress Bar/Skeleton、空状态页）、数据展示类（列表/表格、卡片、树形控件）。

### 可用性评估方法

| 方法 | 最适合发现 | 成本 |
|------|------|------|
| 启发式评估 | 违反已知原则的设计缺陷 | 低（3-5 名专家） |
| 有声思维测试 | 用户真实迷惑点与心理模型偏差 | 中 |
| A/B 测试 | 量化两个方案的效果差异 | 高 |
| 眼动追踪 | 用户忽视的重要元素 | — |

### 可访问性（Accessibility / A11y）

> **WCAG 四大原则（POUR）**：可感知（图片 alt、对比度≥4.5:1）、可操作（键盘可访问、无时限）、可理解（语言清晰、错误可修复）、健壮性（兼容屏幕阅读器）。
> 常见无障碍问题：不能仅靠颜色传达信息、Tab 可遍历、字号≥16px、动画提供关闭选项。

## PART 8a：AI 时代的人机交互

### 范式转变：从 GUI 到对话式 AI

| 维度 | 传统 GUI | AI/LLM 时代 |
|------|------|------|
| 适应方向 | 用户学习软件词汇 | 软件理解用户自然语言意图 |
| 交互单位 | 点击、表单字段 | 多轮对话、自然语言指令 |
| 结果可预期性 | 相同操作→相同结果 | 相似 prompt→结果可能不同 |
| 错误形式 | 明确错误码 | 幻觉难以察觉 |

> HCI 的根本问题没变（更高效/自然/安全的人机交互），只是"计算机"的形态变了。

**提示词工程 = 对话即界面**：Prompt 四要素（角色 Role、上下文 Context、任务 Task、格式 Format）。好的 prompt 结构 = 好的对话界面设计，提示词工程是面向 AI 的 UX 设计。

**LLM 界面的 HCI 新挑战**：
- **响应时间感知**：传统 Shneiderman 标准已不适用（LLM 耗时 3~30s）。**流式输出（Streaming）**逐字呈现大幅降低感知延迟；新模式：骨架屏、打字动画、进度提示。
- **幻觉与信任校准**：界面需展示置信度与引用来源，用户教育"AI ≠ 搜索引擎"。
- **可控性与安全感**：用户随时能中断/撤销，权限边界可见。

### 人机协作的三种模式

| 模式 | 特点 | 典型产品 | 设计重点 |
|------|------|------|------|
| Copilot 副驾驶 | AI 实时建议，人主导接受/拒绝 | GitHub Copilot、Cursor | 低打扰内联建议 |
| Chat 对话 | 多轮自然语言逐步细化 | ChatGPT、Claude | 上下文可见性 |
| Agent 代理 | AI 自主规划执行多步任务 | Claude Code、Devin | Human-in-the-loop |

> **Human-in-the-loop 原则**：关键决策节点暂停确认、AI 行动前展示计划(Plan)、逐步授权、提供审计日志。

**AI 界面新原则**：透明性、可解释性、渐进式信任、人工覆盖权、优雅降级。
> 传统 HCI：人如何适应界面；AI 时代：界面如何理解人。

## PART 8b：前沿研究专题

- **GenUI（生成式界面）**：LLM 主动生成可交互 UI 来回应查询（模块化布局、视觉锚点、可交互控件）。在人类偏好评估中领先传统对话界面 72%。
- **A2UI（Agent-to-User Interface, Google 2025）**：声明式 JSON 流式协议，让 AI Agent "说 UI 语言"。安全（只能引用预批准组件，无法注入脚本）、LLM 友好（扁平化流式）、跨平台（同一载荷渲染为 Angular/Flutter/React/SwiftUI）。
- **DiSCo（流式服务优化）**：两个时延指标 TTFT（首 token 延迟，<200ms 感知"即时"）、TBT（token 间隔）。新 UI 模式：Skeleton Screen、Progressive Disclosure、Token-level streaming、中断控件。
- **GUI Agent（AI 自主操控界面）**：三要素 Grounding（指令→屏幕坐标）、Action（点击/输入）、Reasoning（任务分解）。OSWorld 基准：2024 初 ~12%（GPT-4V）→ 2024 末 ~38%（Claude Computer Use）→ 2025 中 72.6%（Agent S，首超人类基线 72.4%）。专业软件（ScreenSpot-Pro）仍仅 18.9%。
- **AI 辅助 GUI 设计**：自然语言→设计稿（Magic Patterns）、设计稿→代码（Design2Code、Screenshot-to-Code）。Figma 2025：1/3 设计师已在工作中用 AI。

## 模块总结

> 好的界面设计，是让用户忘记自己在使用计算机。
> **AI 时代的好设计，是让计算机理解人，而不是让人迁就计算机。**
